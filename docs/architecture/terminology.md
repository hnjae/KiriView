# Terminology

This document defines recurring runtime terms so architecture, implementation, tests, and specifications use `document` only for the top-level and leaf runtime document concepts. Product behavior remains specified under `../spec/`.

## Core Terms

| Term                            | Meaning                                                                                                                                                                                                                                                                                 | Primary owner                                                | Usage guidance                                                                                                                                                                                                                         |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Document session                | The top-level open state for the window: empty, image mode, or video mode. It owns public mixed-media projections such as active navigation, title subject, zoom readout, and direct media cursor.                                                                                      | Document-session runtime and state owner                     | Use `document session` when behavior spans image and video modes or decides which leaf runtime is active.                                                                                                                              |
| Image document                  | The image-mode leaf runtime and QML facade. It owns image loading, image-document page navigation, opened collection page lists, presentation state, rendering inputs, and image-mode removal fallback facts.                                                                           | Image-document runtime and facade                            | Use `image document` for image-mode internals even when the source is an ordinary direct image. Do not use it to mean archives or directories.                                                                                         |
| Image presentation runtime      | The image-document presentation owner that holds active mode, reading direction, transition phase, zoom, rotation, viewport frame, display-source projections, page visibility, restoration snapshots, and display-refinement demand for the current presentation.                      | Image presentation runtime                                   | Use this term for the single mutable owner of active presentation state. Do not use page-surface objects for zoom, rotation, viewport, visible-rect, display-source, or render-policy ownership.                                       |
| Committed image presentation    | The image presentation that is currently authoritative for visible provider-backed image output, viewport geometry, zoom, pan, rotation, and display-source projection.                                                                                                                 | Image presentation runtime with page-surface resource owners | Use this when pending navigation or pending detail work exists but the previous accepted image remains the visual owner. Do not use it as a synonym for active navigation selection.                                                   |
| Pending image navigation target | A selected image target inside the current active navigation scope whose image resources have not yet become the committed image presentation.                                                                                                                                          | Image-document navigation and presentation workflow owners   | Use this for same-scope image-to-image navigation that has selected a target before display commit. Do not route this as top-level source replacement unless the active scope changes.                                                 |
| Display source projection       | The presentation-owned facts that a visual page presenter consumes: provider URL, revision, source identity, page role, quality/status, source-size hint, load-acknowledgment requirement, retention eligibility, geometry, and rotation.                                               | Image presentation runtime                                   | Use this for the provider-backed visual boundary. Do not use it for visual tile draw entries or as a synonym for source URL.                                                                                                           |
| Display image store             | The C++ memory-pressure owner for immutable provider image entries used by the main display path. It owns entry ids, byte cost, priority, pin leases, original and raster sizes, quality state, source identity, page or animation role, and release.                                   | Display image store boundary                                 | Use this for provider-entry lifetime. Do not make QML engine cache, page-surface objects, or providers themselves the freshness owner.                                                                                                 |
| Image page surface owner        | A page-slot resource owner for static-image payloads, display entries, animation playback, image revision, display-image pin leases, previous-frame retention, predecode cacheability facts, and load lifetime.                                                                         | Primary and secondary page-surface owners                    | Use this term for per-page resources only. Page-surface owners do not expose zoom, rotation, viewport frame, visible rect, display-source policy, render-snapshot policy, tile caches, render resources, or transition state.          |
| Video document                  | The video-mode leaf runtime and QML facade. It owns video source preparation or source-device acceptance, playback state, video status, and video zoom.                                                                                                                                 | Video-document runtime and facade                            | Use `video document` for video-mode internals only.                                                                                                                                                                                    |
| Image document source scope     | The grouped image-document projection category for scope-derived public values. It covers direct media scopes and opened collection scopes.                                                                                                                                             | Image-document public projection owner                       | Use this only for grouped image-document signals or shared projection logic that reacts to either direct media or opened collection scope changes. Do not use it as user-facing wording or when only one concrete scope kind is meant. |
| Direct media scope              | A navigation and source scope for ordinary direct image and direct video URLs, including KDE archive-entry URLs opened as individual media items. This is the architecture term for the ordinary direct media URL scope described in specifications.                                    | Direct media navigation and routing runtimes                 | Use `direct media` when sibling navigation may include both images and videos and the user did not directly open an archive or directory as a collection.                                                                              |
| Opened collection scope         | The internal shared location scope created by directly opening an archive or directory. It supplies media listing, page navigation, opened-collection video routing, predecode, and zoom scope identity.                                                                                | Opened collection scope owner                                | Use `opened collection scope` for behavior that handles directly opened archive collections and directory collections uniformly. Avoid unqualified scope names that imply the archive or directory is itself a document.               |
| Archive collection              | An opened collection scope created by directly opening a supported local archive file. Navigation candidates are supported images and supported videos inside that archive, and some video entries may be playable when the collection access backend exposes a playback source device. | Collection access layer                                      | Use `archive collection` only for directly opened archives, not for KDE archive-entry URLs such as `zip://...!/page.png`.                                                                                                              |
| Directory collection            | An opened collection scope created by directly opening a local directory. Navigation candidates are supported images and supported videos inside that directory tree.                                                                                                                   | Collection access layer                                      | Use `directory collection` when a directory is treated as a page set. Avoid calling this an archive in new user-facing text.                                                                                                           |
| Collection access backend       | The implementation path used to list or read an opened collection.                                                                                                                                                                                                                      | Collection access layer                                      | Use backend-specific wording only when backend choice is observable or implementation-relevant. Do not replace the user-facing collection kind with the backend name.                                                                  |
| Media entry source              | Backend plumbing that lists supported media entries, reads image entry bytes, and creates eligible video playback source devices for an opened collection scope.                                                                                                                        | Collection access layer                                      | Use `media entry source` for backend/runtime ownership of opened collection access. Do not use `document session` for this layer.                                                                                                      |
| Comic book archive              | A directly opened archive with comic-book semantics. It enables comic-only behavior such as Two-Page Spread and Right-to-Left Reading availability.                                                                                                                                     | Comic archive policy owner                                   | Use `comic book archive` when feature availability is intentionally narrower than all archive collections.                                                                                                                             |
| General archive                 | A directly opened non-comic archive treated as an opened collection without comic-only feature availability.                                                                                                                                                                            | General archive policy owner                                 | Use `general archive` when archive collection navigation is active but comic-only controls remain unavailable.                                                                                                                         |
| Displayed image location        | The normalized identity of the currently displayed image plus optional opened collection scope context.                                                                                                                                                                                 | Location identity owner                                      | Use `displayed image location` for the specific image being shown. Do not use it as the source of top-level direct media session identity.                                                                                             |
| Source URL                      | The URL requested by the user, command line, action, or routing operation. It may be a direct media URL, archive file URL, directory URL, or empty URL.                                                                                                                                 | Routing and source-load owners                               | Use `source URL` for the requested input before routing and resolution decide the active runtime context.                                                                                                                              |
| Displayed URL                   | The concrete image URL currently displayed by the image document. For opened collection scopes, this is an internal page URL, not necessarily the original source URL.                                                                                                                  | Image-document state owner                                   | Use `displayed URL` only for the active image payload.                                                                                                                                                                                 |
| Current media target            | The target exposed to Open With and information-panel file actions for the accepted media item. It may be an original direct media URL, an opened directory entry, an opened archive entry URL, or unavailable.                                                                         | Displayed-media operation planning owner                     | Keep this separate from deletion target and source URL. It is an operation target derived from the committed mixed-media session snapshot and source capabilities.                                                                     |
| Deletion target                 | The target used for Move to Trash and Delete Permanently. It may be the displayed direct media URL, the original direct video URL, the opened archive file, the opened directory, or unavailable.                                                                                       | Displayed-media operation planning and deletion runtime      | Keep this separate from current media target. Collection entries can expose a current media target while deletion still targets the containing opened collection.                                                                      |
| Sibling archive navigation      | Navigation between sibling comic book archive files, not navigation inside the current opened collection scope.                                                                                                                                                                         | Sibling archive navigation owner                             | Keep this separate from opened collection page navigation.                                                                                                                                                                             |

## Naming Rules

- Avoid unqualified `document` in new identifiers unless the surrounding type is already unambiguous. Prefer `documentSession`, `imageDocument`, `videoDocument`, `directMediaScope`, `openedCollectionScope`, `archiveCollection`, `directoryCollection`, or `mediaEntrySource`.
- Use `session` for the top-level open state and cross-mode public projections.
- Use `scope` for the set of URLs or pages that navigation, zoom reset, predecode, or toolbar visibility is derived from.
- Use `location` for normalized identity value objects and avoid adding workflow state to location types.
- Do not call KDE archive-entry URLs archive collections. They are direct media URLs unless the user opened the archive file itself.
- Internal shared opened-collection scope identity must spell out opened collection behavior. New public API names must not expose an archive-only or directory-only name when they handle both collection kinds.
- Keep backend-specific names below opened collection access APIs unless behavior really depends on KArchive, libarchive, or directory traversal.
- Grouped signal or API names must spell out their grouping; public state names must prefer concrete terms such as direct media scope or opened collection scope.
- Reserve sibling archive navigation names for sibling comic book archive navigation.
