# Terminology

This document defines recurring runtime terms so architecture, code, tests, and specifications use `document` only for the top-level and leaf runtime document concepts. Product behavior remains specified under `../spec/`.

## Core Terms

| Term                     | Meaning                                                                                                                                                                                                                                              | Primary owner or current names                                                                                                                         | Usage guidance                                                                                                                                                                                                           |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Document session         | The top-level open state for the window: empty, image mode, or video mode. It owns public mixed-media projections such as active navigation, title subject, zoom readout, and direct media cursor.                                                   | `KiriDocumentSession`, `src/session/`, `DocumentSessionState`                                                                                          | Use `document session` when behavior spans image and video modes or decides which leaf runtime is active.                                                                                                                |
| Image document           | The image-mode leaf runtime and QML facade. It owns image loading, image-internal page navigation, opened collection page lists, presentation state, rendering inputs, and image deletion behavior.                                                  | `KiriImageDocument`, `src/document/`, `ImageDocumentRuntime`                                                                                           | Use `image document` for image-mode internals even when the source is an ordinary direct image. Do not use it to mean archives or directories.                                                                           |
| Video document           | The video-mode leaf runtime and QML facade. It owns video source resolution, playback state, video status, and video zoom.                                                                                                                           | `KiriVideoDocument`, `src/video/`, `VideoDocumentRuntime`                                                                                              | Use `video document` for video-mode internals only.                                                                                                                                                                      |
| Media scope              | The grouped image-document projection category for scope-derived public values. It covers direct media scopes and opened collection scopes.                                                                                                          | `mediaScopeChanged`, `ImageDocumentPublicSignal::MediaScope`, `ordinaryDirectMediaScopeActive`, `openedCollectionScopeActive`                          | Use `media scope` for grouped signals or helper logic that reacts to either direct media or opened collection scope changes. Do not use it as user-facing wording or when only one concrete scope kind is meant.         |
| Direct media scope       | A navigation and source scope for ordinary direct image and direct video URLs, including KDE archive-entry URLs opened as individual media items. This is the architecture term for the ordinary direct media URL scope described in specifications. | `DirectMediaCursor`, `ordinaryDirectMediaScopeActive`, media candidate runtimes, direct image/video route plans, `PredecodeScopeKind::DirectMedia`     | Use `direct media` when sibling navigation may include both images and videos and the user did not directly open an archive or directory as a collection.                                                                |
| Opened collection scope  | The internal shared location scope for page-oriented image-document work created by directly opening an archive or directory. It supplies page listing, page navigation, predecode, and zoom scope identity.                                         | `OpenedCollectionScopeLocation`, `OpenedCollectionScopeKind`, `openedCollectionScopeActive`                                                            | Use `opened collection scope` for behavior that handles directly opened archive collections and directory collections uniformly. Avoid unqualified scope names that imply the archive or directory is itself a document. |
| Archive collection       | An opened collection scope created by directly opening a supported local archive file. Candidate pages are supported images and supported videos inside that archive.                                                                                | `OpenedCollectionScopeLocation` with `OpenedCollectionScopeKind::ComicBookArchive` or `OpenedCollectionScopeKind::GeneralArchive`, `MediaEntrySource*` | Use `archive collection` only for directly opened archives, not for KDE archive-entry URLs such as `zip://...!/page.png`.                                                                                                |
| Directory collection     | An opened collection scope created by directly opening a local directory. Candidate pages are supported images and supported videos inside that directory tree.                                                                                      | `OpenedCollectionScopeLocation` with `OpenedCollectionScopeKind::Directory`, `MediaEntrySource*`                                                       | Use `directory collection` when a directory is treated as a page set. Avoid calling this an archive in new user-facing text.                                                                                             |
| Media entry source       | Backend plumbing that lists supported media entries and reads image entry bytes for an opened collection scope. Implementations may use KArchive, libarchive, or directory traversal.                                                                | `MediaEntrySource`, `MediaEntrySourceStore`, `KArchiveMediaEntrySource`, `LibArchiveMediaEntrySource`, `DirectoryMediaEntrySource`                     | Use `media entry source` for backend/runtime ownership of opened archive or directory access. Do not use `document session` for this layer.                                                                              |
| Comic book archive       | A directly opened archive with comic-book semantics, such as CBZ, CBT, CB7, or CBR. It enables comic-only behavior such as Two-Page Spread and Right-to-Left Reading availability.                                                                   | `OpenedCollectionScopeKind::ComicBookArchive`, comic archive policies                                                                                  | Use `comic book archive` when feature availability is intentionally narrower than all archive collections.                                                                                                               |
| General archive          | A directly opened ZIP, TAR, 7Z, RAR, or similar non-comic archive treated as an opened collection without comic-only feature availability.                                                                                                           | `OpenedCollectionScopeKind::GeneralArchive`                                                                                                            | Use `general archive` when archive collection navigation is active but comic-only controls remain unavailable.                                                                                                           |
| Displayed image location | The normalized identity of the currently displayed image plus optional opened collection scope context.                                                                                                                                              | `DisplayedImageLocation`                                                                                                                               | Use `displayed image location` for the specific image being shown. Do not use it as the source of top-level direct media session identity.                                                                               |
| Source URL               | The URL requested by the user, command line, action, or routing operation. It may be a direct media URL, archive file URL, directory URL, or empty URL.                                                                                              | `sourceUrl`, route plans, source-load requests                                                                                                         | Use `source URL` for the requested input before routing and resolution decide the active runtime context.                                                                                                                |
| Displayed URL            | The concrete image URL currently displayed by the image document. For opened collection scopes, this is an internal page URL, not necessarily the original source URL.                                                                               | `displayedUrl`, `DisplayedImageLocation::imageUrl()`                                                                                                   | Use `displayed URL` only for the active image payload.                                                                                                                                                                   |
| Container navigation     | Navigation between sibling comic book archive files, not navigation inside the current opened collection scope.                                                                                                                                      | `containerNavigation*`, Previous Archive, Next Archive                                                                                                 | Keep this separate from opened collection page navigation.                                                                                                                                                               |

## Naming Rules

- Avoid unqualified `document` in new identifiers unless the surrounding type is already unambiguous. Prefer `documentSession`, `imageDocument`, `videoDocument`, `directMediaScope`, `openedCollectionScope`, `archiveCollection`, `directoryCollection`, or `mediaEntrySource`.
- Use `session` for the top-level open state and cross-mode public projections.
- Use `scope` for the set of URLs or pages that navigation, zoom reset, predecode, or toolbar visibility is derived from.
- Use `location` for normalized identity value objects and avoid adding workflow state to location types.
- Do not call KDE archive-entry URLs archive collections. They are direct media URLs unless the user opened the archive file itself.
- Use `OpenedCollectionScopeLocation` for internal shared opened collection scope identity. New public API names should spell out opened collection behavior instead of exposing an archive-only or directory-only name.
- Grouped signal names such as `mediaScopeChanged` may use `media scope`; new public state names should prefer concrete terms such as `ordinaryDirectMediaScope` or `openedCollectionScope`.
- Keep `containerNavigation*` exclusively for sibling comic book archive navigation.
